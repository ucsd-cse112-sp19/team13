Brief Documentation

Here is our brief introduction on the list of our available web component and how to use them

CoreHelloElement

This tag will generate a simple greeting component.

<core-hello></core-hello>

Below is the list of available attributes

  1. Language

    Changes the language of the greeting. By default it is "en".

    <core-hello lang="en"></core-hello>

    Language values

    Accepted ValuesDescriptionColumn
    enEnglish language
    esSpanish language
    jpJapanese language
    frFrench language
  1. Rainbow

    Makes the name to be colorfully animated. By default it is false.

    <core-hello rainbow></core-hello>
  1. Name

    Sets the string that will be displayed after the greeting. By default it is "World".

    <core-hello name="Peter"></core-hello>

CoreSliderElement

This tag will generate a slider component

<core-slider></core-slider>

Below is the list of available attributes.

  1. min

    Change the minimum value of the slider. By default it is 0.

    <core-slider min="0"></core-slider>
  1. max

    Change the maximum value of the slider. By default it is 100.

    <core-slider max="10"></core-slider>
  1. step

    Change the amount of each increment of the slider. By default it is 1.

    <core-slider step="10"></core-slider>
  1. value

    Change the current value of the slider. If the value is past either min or max bounds, it will be corrected on the slider to prevent overflow. However, it will not change the attribute itself. By default it is 0.

    <core-slider value="50"></core-slider>
  1. disabled

    Change the status of the slider to disabled. By default it is false.

    <core-slider disabled></core-slider>
  1. vertical

    Change the slider into a vertical position. By default it is false.

    <core-slider vertical></core-slider>
  1. rainbow

    Change the color of the slider bar into rainbow. By default it is false.

    <core-slider rainbow></core-slider>
  1. color

    Change the primary color of the slider. By default it is white.

    <core-slider color="var(--primary, blue)"></core-slider>

CoreTooltipElement

This component will generate a tooltip, an explanatory boxes that will appear for another element. By default, the target element is its parent, but this can be changed.

<a>
	I am a link.
	<core-tooltip>That is a link.</core-tooltip>
</a>

Below is the list of available attributes

  1. content

    Change the content of the tooltip. By default it is "".

    <core-tooltip content="Description"></core-tooltip>

    You can similarly achieve this by adding the text into the tag.

    <core-tooltip>Description</core-tooltip>

    Or you can add via <slot>.

    <core-tooltip>
	<span slot="tooltip-content">Description</span>
</core-tooltip>
  1. open-delay

    Change the opening delay of the tooltip. To access via JavaScript, the property name is "openDelay". By default, the value is 0.

    <core-tooltip open-delay="1000"></core-tooltip>
  1. close-delay

    Change the closing delay of the tooltip. To access via JavaScript, the property name is "closeDelay". By default, the value is 1000.

    <core-tooltip close-delay="1000"></core-tooltip>
  1. focusable

    Whether the tooltip will open when the target element is focused by tab-index. It should be used with the "for" attribute to set the target element to wait for focus. By default, this is false.

    <div>
	<input type="text" id="text-input"/>
	<core-tooltip for="text-input" focusable>Your Name</core-tooltip>
</div>
  1. for

    Sets the target element by id to listen for focus events. This should be used with the "focusable" attribute to enable or disable its functionality. By default, this is "".

    <div>
	<core-tooltip for="some-target">Phone Number</core-tooltip>
	<input type="number" id="some-target"/>
</div>

  1. placement

    Change the direction in which the tool tip shows up. By default, this is "top".

    <core-tooltip placement="left"></core-tooltip>

    Placement values

    Accepted ValuesDescription
    leftThe tool tip will appear on the left of the target.
    rightThe tool tip will appear on the right of the target.
    topThe tool tip will appear on the top of the target.
    bottomThe tool tip will appear on the bottom of the target.
  1. offset

    Change the offset of the tool tip above the component. By default, the offset is "0".

    <core-tooltip offset="10"></core-tooltip>
  1. no-visible-arrow

    Hide the arrow on the tooltip. By default, the value is false.

    <core-tooltip no-visible-arrow></core-tooltip>
  1. effect

    Change the effect between dark or light. By default, the value is "light".

    <core-tooltip effect="dark"></core-tooltip>

    Effect values

    Accepted ValuesDescription
    darkChange the theme into dark
    lightChange the theme into light
  1. disabled

    Disables the tooltip to not receive any input. By default, it is false.

    <core-tooltip disabled></core-tooltip>
  1. manual

    Relinquishes control of the tooltip. Now it will not do anything on hover or focus. This way you can manually call "open()", "close()", or "toggle()" from a separate callback. By default, this is false.

    <core-tooltip manual></core-tooltip>

CoreLinkElement

This component will generate a link component.

<core-link>This is a link to something special.<core-link>

Below is the list of available attributes

  1. href

    Specifies the url to go to once clicked. By default, it is "".

    <core-link href="www.peter.com"></core-link>
  1. type

    Change the type of the link. In effect, this changes the color scheme. The themes are set by CSS variables, and therefore can be changed. By default, the type is "".

    <core-link type="danger"></core-link>

    Type Value

    Accepted Attribute ValueDescription
    primaryprimary color (blue)
    successsuccess color (lime)
    warningwarning color (orange)
    dangerdanger color (tomato)
    infoinfo color (slategray)
  1. underline

    Gives underline to the link. By default, there is false.

    <core-link underline></core-link>
  1. disabled

    Disables the link. By default, it is false.

    <core-link disabled></core-link>
