Unit Testing

Background

There are 3 technologies used in unit testing:

  • Mocha - JavaScript Test Framework
    • Purpose:
      • Organize all the tests
  • Chai -¬†Assertion Library
    • Purpose
      • Allow programmers to use easier and more intuitive assert statements
  • Showroom - Test Launcher
    • Purpose:
      • Launches a browser and runs the unit tests in the browser
    • Observations:
      • Showroom is just a thin layer built on top of Puppeteer (A node library which enables the Test Launch feature of Showroom)
      • May be helpful to look at Puppeteer documentation if Showroom‚Äôs is too scarce

QA HANDGUIDE

SHOWROOM AND MOCHA 101

File requirements

  1. .showroom folder
  1. File descriptor file inside .showroom folder
  1. Testing file inside test directory
  1. Source file

Format for file descriptor - REQUIRED

export default {
 component: 'component-name',
 path: '../src/component-name.js',
 outerHTML:`
   <component-name>
     content
   </component-name>
 `
}

NameDefinition
componentName of the component
pathDirectory of the component class
outerHTMLRequired for `showroom` command and make it shows in the browser
ADDITIONAL PROPERTIES
descriptionDescription of component usage
eventsUsed for component that utilizes dispatchEvent() method
propertiesThe component‚Äôs properties
attributesThe components attributes

Format for testing file

const { assert } = require('chai');
const showroom = require('showroom/puppeteer')();

/**
 * Main testing method using mocha
 */

describe('component-name', () => {
  /**
   * Starting showroom and puppeteer in order to create the html document
   * to be tested
   */
  before(async () => {
    await showroom.start();
  });

  /**
   * Stopping showroom and puppeteer
   */
  after(async () => {
    await showroom.stop();
  });


  /**
   * Creating new component for each test case based on the descriptor
   */
  beforeEach(async () => {
    await showroom.setTestSubject('component-name');
  });

  /**
   * This test checks ....
   */
  it('Should be the result', async () => {
    assert.equal(true, true);
  });
});

Important Mocha, Showroom and Puppeteer Function:

const { assert } = require('chai');const showroom = require('showroom/puppeteer')();

Showroom/puppeteer

  1. showroom.setTestSubject(componentName:string):Promise<JSHandle>
  1. showroom.setProperty(propertyName:string, value: any):Promise
  1. showroom.getProperty(propertyName:string):Promise<any>
  1. showroom.setAttribute(attributeName:string, value: string):Promise
  1. showroom.getAttribute(attributeName:string):Promise<string>
  1. showroom.hasAttribute(attributeName:string):Promise<boolean>
  1. showroom.removeAttribute(attributeName:string):Promise
  1. showroom.find(path:string):Promise<JSHandle> - search DOM within the component, use double slash // for shadow-DOM penetration (DOM from src, not outerHTML)
  1. showroom.getEventList():Promise<SerializedEventObject[]> - search event from file descriptor
  1. showroom.getLastEvent():Promise<SerializedEventObject>

Note: await is required, so every mocha function must be async because showroom function returns promise
