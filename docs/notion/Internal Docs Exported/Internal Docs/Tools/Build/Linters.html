Linters

Currently, we have two linters running on all .html, .css, and .js files found recursively through the project directory. I’m still trying to find an HTML linter, but I am not sure any exist. Please let me know if you can find one…. There’s this one https://www.npmjs.com/package/html-linter but it has very few weekly downloads so I’m a little worried. If there is enough support for it, it would be easy to implement.

eslint: We are currently using a few plugins (airbnb-base and html) for this, all reflected in the eslintrc.json config file. Eslint lints javascript code. It will look at all of your javascript files and your embedded scripts in your html files, and get mad at you for things like unused variables.

Simply install the Eslint plugin if you want it to give you real-time feedback in VSCode. Also, if you want it to look into your HTML files as well, add

"eslint.validate": [ "javascript", "html" ]

to your settings JSON in VSCode.

stylelint: stylelint looks at CSS. It searches your CSS files and your embedded stylesheets in your html. It will get mad at you for things like duplicate selectors or empty brackets. It also supports SCSS and LESS linting, but I will only add this functionality if it is requested since we don’t want to add things we don’t need.

To use stylelint real-time linting in VSCode, install this plugin “https://marketplace.visualstudio.com/items?itemName=shinnn.stylelint" . As per the documentation, you might want to add these lines to your settings JSON :

"css.validate": false,
"less.validate": false,
"scss.validate": false,

This basically tells the default VSCode linter and stylelint to not interfere with each other and give duplicate issues. That way, you won’t be surprised if Travis’ linter is not a fan of your code but VSCode says it’s fine.
