Purpose: research tools and best practices for testing web components

Test Driven Development:

  1. Writing tests for the required software functionality
  1. Run the tests for the software functionality
  1. Implement the software functionality
  1. Fix bugs and refactor until all tests pass
  1. Repeat the cycle for new functionality

Broadly goes over what to test in Front-end components:

  1. HTML validation and accessibility testing
    1. As Professor mentioned, we can write malformed HTML that will visually look the same as valid HTML in the browser. To avoid this, have each .html file go through a validator to make sure it is valid. The W3C provides a validator we can use at https://validator.w3.org/
    1. We can also do Web Accessibility evaluation to see if the components meet certain accessibility guidelines. Depending on what accessibility criteria we impose upon ourselves, we can search for a tool that fits our needs using https://www.w3.org/WAI/ER/tools/ which provides a list of tools to examine.
  1. CSS Validation
    1. Same as with HTML, we can ensure all CSS files have the correct syntax using W3C validation service at https://jigsaw.w3.org/css-validator/
  1. JavaScript Unit testing
    1. Introduces unit testing for JS and a number of tools to do the unit testing such as Karma,UnitJS, and Jasmine to name a few.

Why use RequireJS?

  • It is an open source JavaScript library under the MIT license.
  • It provides asynchronous module loading.
  • It has the ability to load nested dependencies.
  • If you have many small files, then don't need to worry about keeping track of dependencies order.
  • It provides support for plugins and loads more than one JavaScript files.

Features of RequireJS

  • It manages the dependencies between JavaScript files and improves the speed and quality of the code.
  • It combines and minifies the modules into one script for an optimized experience.
  • It reduces code complexity in large applications.
  • It gathers different JavaScript files from different modules at the time of compilation.
  • It allows for easy debugging as it loads the files from plain script tags.
